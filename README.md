# AskifyPDF

AskifyPDF is an intelligent, evidence-first document analysis platform designed for students and researchers. It allows users to query complex PDF documents (such as research papers) using a Retrieval-Augmented Generation (RAG) pipeline, providing highly accurate AI answers grounded directly in the source text. 

Built with an uncompromising "Architectural Monolith" design philosophy, AskifyPDF offers a distraction-free, dense, and fully synchronized workspace where every AI citation acts as a clickable navigation bridge directly to the relevant PDF page.

## Features

- **Evidence-First AI Chat**: Engage with research papers fluidly. The AI provides answers securely grounded in document context, reducing hallucinations.
- **Precision Page Navigation (Click-to-Jump)**: Citations generated by the AI are interactive buttons (`PAGE_13`). Clicking them instantly synchronizes the PDF viewer to the exact source page.
- **Architectural Monolith UI**: A brutalist, dark-themed, high-density interface focused completely on the document and the AI dialogue. No distracting sidebars, pure focus.
- **RAG Engine with Self-Healing Retrieval**: Powered by Pinecone vector databasing, the backend gracefully handles identifier drift, ensuring citations and context are reliably retrieved even during indexing asynchronous delays.
- **High-Fidelity PDF Rendering**: Built on `react-pdf` with custom magnification, custom scrolling, and library thumb-nailing.
- **Cloud-Native Auth & Storage**: Supabase handles secure user authentication and document object storage.

## Tech Stack

- **Frontend**: React (Vite), Tailwind CSS, Framer Motion, React-PDF, Lucide Icons.
- **Backend**: Python, FastAPI, Langchain, Pinecone (Vector DB), Ollama (Mistral for local inference / LLM).
- **Database & Auth**: Supabase (PostgreSQL, Storage, Auth).

## Getting Started

### Prerequisites
- Node.js (v18+)
- Python (3.10+)
- Supabase account & project
- Pinecone account & index
- Ollama running locally (if using local LLM inference)

### 1. Backend Setup
Navigate to the backend directory, install dependencies, and configure the environment:
```bash
cd app/backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```
Create a `.env` file in the `app/backend` directory:
```env
PINECONE_API_KEY=your_pinecone_api_key
PINECONE_INDEX_NAME=askify-pdf
SUPABASE_URL=your_supabase_project_url
SUPABASE_KEY=your_supabase_anon_key
# Ensure Ollama is running locally if configured for local embeddings/chat
```
Start the backend server:
```bash
python main.py
# or 
uvicorn main:app --reload
```

### 2. Frontend Setup
Navigate to the frontend directory, install dependencies, and configure the environment:
```bash
cd app/frontend
npm install
```
Create a `.env` file in the `app/frontend` directory:
```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_API_URL=http://127.0.0.1:8000
```
Start the frontend development server:
```bash
npm run dev
```

## System Architecture

1. **Document Ingestion**: PDFs uploaded via the React frontend are stored in Supabase Storage.
2. **Vector Indexing**: Background processes extract text, divide it into overlapping semantic chunks (preserving page metadata), and embed them into a high-dimensional vector space stored in Pinecone.
3. **Retrieval-Augmented Generation**: User queries are embedded and mapped against the Pinecone index to find the most contextually relevant chunks.
4. **Synthesis & Citation**: The backend system processes the raw context, generates an intelligent answer using the LLM, and formats clickable citations that are synced via React state directly back into the PDF viewer.
